# Google Maps API キー 取得・設定ガイド（初心者向け）

`/map` ページで地図を表示するために、Google が提供する **Maps JavaScript API** のキーを取得します。所要時間は **約15分**、Googleアカウントとクレジットカードが必要です（無料枠内なら **請求は0円** で済みます）。

---

## 1. なぜキーが必要なのか（30秒で）

- Google Maps はGoogleが提供するクラウドサービス。利用回数を計測するため、利用者ごとに **「鍵（APIキー）」** を発行します。
- このアプリは **その鍵を読み込んで** 地図を表示する仕組みです。
- 月 **28,500回の地図表示までは無料**（2025年時点）。商談での利用なら通常まず超えません。

---

## 2. 取得手順

### Step 1. Google Cloud にログイン

1. ブラウザで <https://console.cloud.google.com/> を開く
2. Googleアカウントでログイン
3. 初回は **「利用規約に同意」** → **国: 日本** を選択して続行

### Step 2. プロジェクトを作る

「プロジェクト」=この鍵を入れておく **箱** のことです。

1. 画面上部の **プロジェクト選択ドロップダウン** をクリック
2. 右上の **「新しいプロジェクト」** をクリック
3. プロジェクト名: `ev-insight` と入力 → **作成**
4. 作成完了後、上部のドロップダウンで `ev-insight` を選択（必ず！）

### Step 3. お支払い情報を登録（無料枠を使うのに必須）

> 心配しないでOK：無料枠を超えるまで請求は発生しません。Googleが「うっかり高額請求」を防ぐ仕組みもあります（Step 7で設定）。

1. 左メニュー → **「お支払い」**
2. **「請求先アカウントを作成」** または **「請求先アカウントをリンク」**
3. クレジットカードと住所を入力 → 完了

### Step 4. 必要なAPIを有効化

「APIライブラリ」で以下の **3つ** をそれぞれ検索→「有効にする」してください：

1. 左メニュー → **「APIとサービス」** → **「ライブラリ」**
2. 検索して有効化:
   - **Maps JavaScript API** — 地図本体の描画（必須）
   - **Places API** — 住所・施設名検索 / 周辺の充電器取得（必須）
   - **Geocoding API** — 住所⇄座標の変換（推奨）
3. （任意）**Map Tiles API** — スタイル付き地図がきれいになる

> いずれも月 $200 分の無料枠を共有しており、商談利用なら通常請求は発生しません。

### Step 5. APIキーを発行

1. 左メニュー → **「APIとサービス」** → **「認証情報」**
2. 上部の **「+ 認証情報を作成」** → **「APIキー」**
3. キーが表示されるので **コピー**（例: `AIzaSyB...で始まる長い文字列`）
4. **「キーを制限」** を必ず押す

### Step 6. キーの利用制限を設定（重要！）

> 制限なしのキーをネットに置くと、悪用されて高額請求につながる可能性があります。**必ず設定**してください。

「APIキーの編集」画面で：

#### アプリケーションの制限

- **「ウェブサイト」** を選択
- 「ウェブサイトの制限」で **「項目を追加」** をクリック
- 以下を1行ずつ追加：
  - `http://localhost:3001/*`（開発用）
  - `http://localhost:3000/*`（念のため）
  - 本番にデプロイしたら本番ドメインも追加 例: `https://ev-insight.example.com/*`

#### APIの制限

- **「キーを制限」** を選択
- ドロップダウンで以下にチェック:
  - **`Maps JavaScript API`**（必須）
  - **`Places API`**（住所検索に必須）
  - **`Geocoding API`**（推奨）
- （Step 4で Map Tiles も有効にしたなら）`Map Tiles API` もチェック

最後に **「保存」**。

### Step 7. 予算アラートを設定（保険）

1. 左メニュー → **「お支払い」** → **「予算とアラート」**
2. **「予算を作成」** → 金額に **`1000円`** を設定
3. しきい値 50% / 90% / 100% でメール通知 → 作成

これで月の請求が1000円に近づくとメールが届きます（無料枠内ならまず届きません）。

### Step 8. （オプション）Map ID を発行

カラフルな最新スタイルの地図を使うために `Map ID` を発行します。なくても動きますが、推奨。

1. 左メニュー → **「Google Maps Platform」** → **「Map management」**（または直接 <https://console.cloud.google.com/google/maps-apis/studio/maps>）
2. **「+ Create Map ID」**
3. 名前: `ev-insight-default`、Map type: **JavaScript**、Vector を選択 → 保存
4. 一覧画面に戻ると **Map ID（英数字の文字列）** が表示される → コピー

## 3. このプロジェクトに鍵を設定する

ターミナルでプロジェクトに移動済みの状態で：

### 3-1. `.env.local` ファイルを作る

プロジェクト直下（`package.json` と同じ階層）に **`.env.local`** という名前のファイルを作り、以下を貼り付けます：

```
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=ここにStep5のキーを貼る
NEXT_PUBLIC_GOOGLE_MAPS_MAP_ID=ここにStep8のMap IDを貼る
```

例：

```
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=AIzaSyBxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NEXT_PUBLIC_GOOGLE_MAPS_MAP_ID=a1b2c3d4e5f6g7h8
```

> **注意**：イコール `=` の前後にスペースを入れない。値はクォーテーション不要。

> Map ID をスキップした場合は `NEXT_PUBLIC_GOOGLE_MAPS_MAP_ID` の行は削除してOK（デフォルト値 `ev-insight-map` が使われます）。

### 3-2. dev サーバを再起動

`.env.local` の変更は **再起動しないと反映されません**。

```bash
# 既存のサーバを止めて
# （ターミナルで Ctrl + C）

# もう一度起動
npm run dev
```

### 3-3. ブラウザで確認

<http://localhost:3001/map> を開く → **地図が表示されればOK**。

## 4. うまく行かないときのチェックリスト

| 症状 | 原因と対処 |
|---|---|
| 地図が灰色で「For development purposes only」 | 課金（お支払い）が未登録。Step 3 をやり直し |
| 「Google Maps API キーが未設定です」と画面に出る | `.env.local` のキー名のスペルミス、または dev サーバ再起動忘れ |
| 真っ白で何も出ない | ブラウザのコンソール（F12）でエラーを確認。`RefererNotAllowed` なら Step 6 のドメイン制限を見直し |
| `ApiNotActivatedMapError` | Step 4 の API有効化忘れ |
| 地図は出るがピンが出ない | Map ID が無効。`.env.local` から `MAP_ID` 行を削除して再起動で動作するはず |

---

## 5. セキュリティの心得

- `.env.local` は **絶対にGitHubにアップしない**（Next.jsデフォルトの`.gitignore`で除外されています）
- キーをSlackやメールで共有しない
- もし漏れた疑いがあれば、Step 5 の認証情報画面でキーを **削除して新しく作り直す**

---

これで設定完了です。営業先の住所周辺に地図中心を移したい場合は、`lib/map-data.ts` の `REGION_PRESETS` に新しいエリアを追加できます。
